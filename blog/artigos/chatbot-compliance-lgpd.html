Erros Fatais Que Você Precisa Evitar

90% das implementações que falham cometem os mesmos erros. Erro 1: Automatizar processos ruins (automatização de caos gera caos mais rápido). Erro 2: Não treinar a equipe adequadamente (resistência interna mata projetos). Erro 3: Ignorar o feedback dos usuários (a IA precisa evoluir constantemente). Erro 4: Não ter plano de fallback humano (100% automação é receita para desastre). Erro 5: Métricas vagas ou inexistentes (o que não é medido não é gerenciado). Erro 6: Subestimar o tempo de implementação (rushed projects fail). Erro 7: Esquecer conformidade com LGPD/privacidade.

❓ Perguntas Frequentes (FAQ)

1. Chatbot precisa estar em conformidade com LGPD?

SIM, obrigatório. Chatbot coleta dados pessoais (nome, telefone, CPF, e-mail). Requisitos: consentimento explícito, finalidade clara, armazenamento seguro, direito de exclusão, registro de auditoria. Multa LGPD: até R$50 milhões.

2. Como pedir consentimento LGPD no chatbot?

Primeira interação: 'Olá! Para continuar, preciso coletar [dados X] para [finalidade Y]. Você concorda? Digite SIM para aceitar.' + link para política de privacidade. Armazene timestamp e resposta do consentimento.

3. Onde armazenar dados do chatbot de forma segura?

Essencial: criptografia em repouso (AES-256), criptografia em trânsito (TLS 1.3), servidor no Brasil (soberania de dados), acesso restrito (IAM), logs de auditoria, backup criptografado. Considere certificação ISO 27001.

4. Chatbot pode reter dados por quanto tempo?

Apenas pelo tempo necessário. Exemplos: lead não convertido = 6 meses → deletar; cliente ativo = enquanto houver relação comercial; após cancelamento = manter apenas dados fiscais por 5 anos (obrigação legal).

5. Como implementar direito de exclusão no chatbot?

Cliente deve poder pedir: 'quero excluir meus dados'. Chatbot: confirma identidade → deleta dados de todas bases (chatbot, CRM, analytics) → envia confirmação. Prazo legal: até 15 dias. Mantenha registro da exclusão (auditoria).
